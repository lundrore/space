项目

系列项目

tiny: ssloy 的一系列小项目 (tinyrenderer, tinyraytracer, tinyraycaster, tinykaboom, tinycompiler) 的 Go 实现，作者没有使用外部库，因此跟着实现了 obj, tga 文件的读取，数学库用于之后大多数的图形学项目上。 还包括了 implementing-a-tiny-cpu-rasterizer 这个 tinyrasterizer，作者文章只写到了 Part 6，尽管代码仓库进展多一些，还是只跟到了这里

eva: Dmitry Soshnikov 的系列课程 (Building an Interpreter from Scratch, Building a Parser from Scratch, Building a Transpiler from Scratch, Building a Typechecker from Scratch) 的 Python 实现，Javascript -> Python, JS 的代码写的太灵活，用 Go 实现的有些痛苦，就全部使用 Python 来做了

pikuma: gustavopezzi 的系列课程 (3D Computer Graphics Programming, Game Physics Engine Programming, 2D Game Engine Development) 的 Go 实现，游戏引擎部分作者嵌入了 lua 解释器，这一步之后没有向后做

inoneweekend: Ray Tracing in One Weekend 系列 Go 实现，还有文章 Rasterization in One Weekend 的前两部分，用 tiny 中的数学库来代替 glm，glm 的行为有些不同，做了一些修改

书籍代码

computation: computationbook 中译《计算的本质——深入剖析程序和计算机》，关于计算理论介绍的书籍，Ruby -> Go

dsacc: 数据结构与算法分析——C语言描述 数据结构和算法，C -> Go

groundup: Ray Tracing from the Ground Up 实现小型的光线追踪器，C++ -> Go

baseline: compiling to assembly from scratch 编译器，书中解析是由解析组合子完成，完成第一部分，Typescript -> Go

dezero: 深度学习入门2——自制框架 Python -> Go，完成到步骤43 神经网络，numpy 的 broadcasting 使复刻的有些别扭

从头开始加参考

基本是从头开始写，代码结构不同或是没有参考代码

bmp: 一个 bmp 图片格式的解析器，结构体定义和注释主要来自 [MS-WMF]: Windows Metafile Format，

elf: linux 下可执行文件 elf 的解析器，主要参照文档 TIS1.1.pdf 实现， 借鉴了 finixbit/elf-parser 和 macmade/ELFDump，输出样式仿照了 readelf，实现了其 -h, -l, -S, -s 这几个选项

parc: OCaml 写的解析组合子，主要参照 Graham Hutton 书《Programming in Haskell》的第十三章以及作者相关的两篇论文， 另外用该解析组合子实现 wyas，Towards a Standard Library 一章节实现的有些问题

sexp: 一个简单的 s 表达式的解析，打算用于 ucsd compilers s23，参照了项目使用的 Rust 的 sexp 以及 eva 中用到的 s表达式的解析，框架还是来自于 interpreterbook

codecrafters: codecrafters 网站的挑战解，代码没有整理，完成的 shell 和 browser 网站又有更新，有时间继续完成

bencode: 为写 codercrafers 的 bitorrent 做的准备，将前面的手动解析的代码用 marshal/unmarshal 进行包装，方便使用

argparse: 刚开始学 Go 写的简单的命令行参数解析器，在 argparse-rosetta-rs 发现的 pico-args 觉得代码量不大，就自己随便开始写了，逻辑混乱，乱七八糟，参考了 go 的 flag 和 python 的 getopt

重写加上修改

有参照物的代码，但是进行了修改，或是结合多个参考的混合物

automata: 自动机是来自 Dmitry Soshnikov 的自动机课程，解析部分参照 implementing a regular expression engine 结合自己代码生成的 AST

lept: miloyip 的 json-tutorial，依照其步骤，但是采取 interpreterbook 的代码框架进行解析的

chibiccgo: rui314 的 chibicc Go 重写，一个五脏俱全的 C 编译器，后端参考 rvcc 写了 RISC-V 版本，但中途测试失败没有继续，最终完成的版本只有 x86-64 的内容

brainfuck: 一个渐进实现的 brainfuck 解释器，包括几个不同版本

lisgo: norvig lispy 的 Go 重写，文章还有第二篇待做

chip8: 参照这篇文章 Building a CHIP-8 Emulator，指令文档主要参考 devernay 里的资料，做了一些更改使其通过 chip8-test-rom 和 chip8-test-suite 里的大多数测试。没有实现声音

重写的教程

跟随教程，将原语言转写为 Go 实现，没有过多改变

lc3: Write your Own Virtual Machine C -> Go，键盘部分尝试了几种方式，都不完美，最后还是使用了外部库 keyboard

kilo: Build Your Own Text Editor 的 Go 实现，第二章的 raw mode 从 go/term.git 和 go/sys.git 拿了代码

emurv: 参照 PLCT 的 KSCO老师带你手搓RISC-V高性能模拟器 写的，完成到第7课，读取可执行文件部分从之前写的 elf 解析器中拿取了一些代码，指令实现有些问题没有继续向后做

rvemu: 跟着 Writing a RISC-V Emulator in Rust 实现的, 前面的阶段运行正常。最后一步执行 xv6，会运行很长时间然后报错，可能同上也是指令实现问题？我可能需要一个详细的测试

dnsguide: dnsguide Rust -> Go，其实还是不懂，照着翻译了一遍，不过代码可以运行

vulkan-tutorial: vulkan-tutorial C++ -> Go, 完成到27课，需要加载模型。在前面 tinyrenderer 项目中编写的 obj 文件加载缺少了对对象 m 的支持，有时间修改下再继续完成

simpledb: db_tutorial C -> Go, 用 Go 的测试来替代 Ruby 的 rspec 进行测试用例的编写

caster: lodev raycasting C++ -> Go, 使用 SDL2 来替代作者的 QuickCG，最后一章只做了作者提供代码的增补部分

smu: Gottox/smu C -> Golang，支持了 unicode, 本博客使用的 markdown 解析器

lox: craftinginterpreters 前半部分的代码实现，不过卡在了第十二章，代码未整理，只有最近的这个版本。跟随 codecrafters 进行了测试，等有时间重新整理各个章节代码并完整实现后再更新
